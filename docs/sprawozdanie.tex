\section{Opis przedmiotu zlecenia}
%==============================================================================

Przedmiotem zlecenia projektowego jest zaprojektowanie, implementacja i wdrożenie
zintegrowanego systemu informatycznego łączącego \textbf{wypożyczalnię książek}
oraz \textbf{sklep internetowy} pod wspólną marką \textit{meow}.

Zgodnie z wymaganiami prowadzącego przedmiot \emph{Projektowanie i programowanie systemów internetowych I}
system musi udostępniać:
\begin{itemize}
    \item \textbf{interfejs webowy} dla klientów (przeglądanie katalogu, koszyk, checkout,
          rezerwacja egzemplarzy bibliotecznych, konto użytkownika),
    \item \textbf{panel administracyjny} do zarządzania katalogiem, klientami,
          wypożyczeniami i zamówieniami sklepowymi,
    \item \textbf{REST API} (z dokumentacją OpenAPI) jako warstwę danych dla
          planowanej aplikacji mobilnej.
\end{itemize}

\subsection{Zakres funkcjonalny}

System obejmuje m.in.:
\begin{itemize}
    \item rejestrację i logowanie użytkowników (role: klient, administrator),
    \item zarządzanie książkami, egzemplarzami bibliotecznymi i stanem magazynowym sklepu,
    \item proces zakupu (koszyk, dane dostawy, składanie zamówienia),
    \item wypożyczenia (termin 30 dni), rezerwacje, zwroty oraz obsługę opóźnień,
    \item eksport danych raportowych z modułu wypożyczeń (\texttt{ReportsController}, widok finansowy),
    \item integrację z zewnętrznymi usługami (cache Redis, Brevo API / Mailpit, szkielet AI/Ollama).
\end{itemize}

\section{Opis technologiczny proponowanego rozwiązania}
%==============================================================================

\subsection{Architektura ogólna}

Aplikacja oparta jest na wzorcu \textbf{MVC} (Model--View--Controller) w środowisku
\textbf{ASP.NET Core} (.NET~10). Warstwa prezentacji realizowana jest przez widoki
Razor (\texttt{.cshtml}); logika biznesowa znajduje się w kontrolerach i serwisach;
dane utrwalane są w relacyjnej bazie \textbf{MySQL~8} przez \textbf{Entity Framework Core}
(Pomelo provider).

\subsection{Model danych}

Główne encje ORM (\texttt{LibraryDbContext}):
\begin{itemize}
    \item \texttt{Book} --- tytuł, autor, gatunek, ceny, ilość w sklepie,
    \item \texttt{Egzemplarz} --- egzemplarz biblioteczny (numer inwentarzowy, stan),
    \item \texttt{Klient}, \texttt{User} --- dane klienta i konto logowania,
    \item \texttt{Wypozyczenie}, \texttt{Platnosc} --- wypożyczenia i kary,
    \item \texttt{Zamowienie} --- zamówienia sklepowe.
\end{itemize}

Migracje EF Core stosowane są przy starcie aplikacji (\texttt{Database.Migrate()}).

\subsection{Warstwa API i cache}

Publiczne endpointy REST (prefix \texttt{/api}):
\begin{itemize}
    \item \texttt{GET /api/books} --- katalog z opcjonalnym filtrem \texttt{?q=} i \texttt{?gatunek=},
    \item \texttt{GET /api/books/\{id\}} --- szczegóły książki,
    \item \texttt{POST /api/auth/login} --- uwierzytelnienie JWT,
    \item \texttt{POST /api/books/refresh-cache} --- unieważnienie cache (dev/admin).
\end{itemize}

Serwis \texttt{BookCatalogCacheService} buforuje wyniki katalogu w Redisie
(klucze z prefiksem \texttt{meow:}), co ogranicza obciążenie bazy przy częstych
odczytach (np. live search w sklepie).

\subsection{Uwierzytelnianie i autoryzacja}

\begin{itemize}
    \item \textbf{Panel WWW:} sesja po logowaniu (\texttt{HttpContext.Session}),
          atrybut \texttt{[MeowAuthorize(Rola = "Admin")]} na akcjach administracyjnych.
    \item \textbf{API mobilne:} token JWT (HS256, issuer/audience w \texttt{appsettings.json}),
          ważność 8~h, claim \texttt{klient\_id} dla powiązania z profilem.
\end{itemize}

Hasła przechowywane są jako skróty BCrypt.

\subsection{Lokalizacja i interfejs}

Interfejs obsługuje języki \textbf{polski} (domyślny) i \textbf{angielski}.
Przełącznik kultury zapisuje wybór w ciasteczku (\texttt{CookieRequestCultureProvider}).
Teksty UI w plikach \texttt{Resources/SharedResources.*.resx}.

Responsywność (RWD) zapewnia \texttt{wwwroot/css/responsive.css} oraz komponenty Bootstrap~5.

\subsection{Integracje zewnętrzne (stan implementacji)}

\begin{itemize}
    \item \textbf{Redis, Swagger, mailing} --- zaimplementowane. Cache Redis opcjonalny
          (fallback: \texttt{DistributedMemoryCache}). Maile potwierdzeń zamówień:
          lokalnie \textbf{Mailpit} (port 8025), produkcja \textbf{Brevo Transactional API}
          (\texttt{Smtp\_\_ApiKey}, HTTPS --- wymagane na Railway z powodu blokady portu SMTP).
    \item \textbf{Ollama (AI)} --- szkielet (\texttt{AITestController}, \texttt{AIChatController});
          wymaga osobnej instalacji Ollama (integracja w zakresie Kyrylo Shchedrin).
    \item \textbf{Qdrant + ONNX} --- infrastruktura Docker i \texttt{VectorSearchService};
          wykorzystanie produkcyjne opcjonalne.
\end{itemize}

\section{Instrukcja uruchomienia systemu}
%==============================================================================

\subsection{Wymagania wstępne}

\begin{itemize}
    \item Docker Desktop (Windows/macOS) lub Docker Engine + Compose (Linux),
    \item Porty wolne: \texttt{5000} (WWW), \texttt{3307} (MySQL), \texttt{6379} (Redis),
          \texttt{8025} (Mailpit), opcjonalnie \texttt{6333} (Qdrant),
    \item Klon repozytorium:
\begin{lstlisting}[language=bash]
git clone https://github.com/04Sqv3r/s4-ppsi1.git
cd s4-ppsi1
git checkout filip/aktualna-wersja   # lub main po merge
\end{lstlisting}
\end{itemize}

\subsection{Uruchomienie lokalne}

W katalogu projektu:

\begin{lstlisting}[language=bash]
docker compose up --build
\end{lstlisting}

Po zbudowaniu kontenerów:
\begin{itemize}
    \item Strona WWW: \url{http://localhost:5000}
    \item Dokumentacja OpenAPI (Swagger UI): \url{http://localhost:5000/swagger}
    \item Specyfikacja JSON: \url{http://localhost:5000/swagger/v1/swagger.json}
\end{itemize}

\textbf{Pierwsze konto:} zarejestruj użytkownika przez \texttt{/Account/Register},
następnie zaloguj się. Przy starcie aplikacji \texttt{DatabaseSeeder} uzupełnia katalog
książek (jeśli baza jest pusta). Rolę administratora można nadać w bazie:
\begin{lstlisting}[language=sql]
UPDATE Users SET Rola = 'Admin' WHERE Login = 'twoj_login';
\end{lstlisting}
(Użytkownik \texttt{filip} otrzymuje rolę Admin automatycznie, jeśli istnieje w bazie.)

\textbf{Zatrzymanie:}
\begin{lstlisting}[language=bash]
docker compose down
\end{lstlisting}

\subsection{Uruchomienie zdalne (produkcja)}

\subsubsection{Docker Compose (VPS)}

\begin{lstlisting}[language=bash]
docker compose -f docker-compose.yml \
  -f docker-compose.prod.yml up -d --build
\end{lstlisting}

Wymagane zmienne środowiskowe:
\begin{itemize}
    \item \texttt{ConnectionStrings\_\_DefaultConnection}
    \item \texttt{ConnectionStrings\_\_Redis}
    \item \texttt{Jwt\_\_Key} (min. 32 znaki)
    \item opcjonalnie: \texttt{Smtp\_\_ApiKey}, \texttt{Smtp\_\_From} (Brevo) --- patrz \texttt{docs/SMTP.md}
\end{itemize}

\subsubsection{Railway (zalecane dla demo)}

Publiczny adres wdrożenia zespołu:
\url{https://meow-app-production.up.railway.app}

\begin{enumerate}
    \item Utwórz projekt na \url{https://railway.app} (deploy z GitHub lub CLI).
    \item Dodaj usługę \textbf{MySQL} (Redis opcjonalny).
    \item W aplikacji webowej ustaw m.in.:
    \begin{itemize}
        \item \texttt{ASPNETCORE\_URLS=http://0.0.0.0:8080}
        \item \texttt{ConnectionStrings\_\_DefaultConnection} --- z panelu MySQL
        \item \texttt{Jwt\_\_Key} --- min. 32 znaki
        \item \texttt{Smtp\_\_ApiKey} --- klucz Brevo API (\texttt{xkeysib-...})
        \item \texttt{Smtp\_\_From} --- zweryfikowany nadawca w Brevo
    \end{itemize}
    \item Railway wykrywa \texttt{Dockerfile} i buduje obraz automatycznie.
    \item W zakładce \emph{Networking} wygeneruj publiczny URL.
    \item Dokumentacja OpenAPI:
          \url{https://meow-app-production.up.railway.app/swagger}
\end{enumerate}

Szczegóły: plik \texttt{DEPLOY.md} w repozytorium.

\subsection{Pipeline CI}

Przy pushu na gałąź \texttt{main} uruchamiany jest workflow GitHub Actions
(\texttt{.github/workflows/ci.yml}): \texttt{dotnet restore}, \texttt{dotnet build},
\texttt{dotnet publish}.

\begin{lstlisting}[language=bash]

  -d "{\"login\":\"user\",\"haslo\":\"haslo123\"}"
\end{lstlisting}

\begin{lstlisting}[language=bash]

\end{lstlisting}

\textbf{Autoryzacja JWT} (nagłówek):
\begin{lstlisting}
Authorization: Bearer <token>
\end{lstlisting}

\section{Podsumowanie}
%==============================================================================

System \textit{meow} realizuje zintegrowany model biblioteki i sklepu internetowego
z panelem administracyjnym, raportami finansowymi oraz warstwą REST API gotową pod
aplikację mobilną. Rozwiązanie konteneryzowane (Docker) umożliwia jednolite uruchomienie
lokalne i zdalne; dokumentacja OpenAPI dostępna jest przez Swagger UI pod ścieżką
\texttt{/swagger}.

Aplikacja produkcyjna działa pod adresem
\url{https://meow-app-production.up.railway.app}.
Mailing potwierdzeń zamówień zrealizowano przez Brevo Transactional API
(weryfikacja nadawcy w panelu Brevo).

Element opcjonalny w trakcie rozbudowy: pełna integracja Ollama (moduł AI).
